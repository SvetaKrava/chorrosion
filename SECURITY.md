# Security Policy

Thank you for helping keep chorrosion secure. Please use the channels below to report security vulnerabilities.

## Reporting a Vulnerability

- Preferred: Use GitHub's Security Advisories for this repository: open the "Security" tab and create an advisory.
- If you need to share sensitive information privately, open a private issue and mark it as security-related.

When reporting, please include:

- A concise summary of the issue and its impact.
- Steps to reproduce, including sample files or commands if applicable.
- Affected versions/branches and environment details.
- Any suggested mitigations or PoC code (optional).

## Response Process

- We will acknowledge receipt within 3 business days.
- We will work with you to triage, and where appropriate, prepare and ship a fix in coordination with you.
- We aim to publish a public advisory or merge a fix within a reasonable timeframe depending on severity.

## Supported Versions

We support `main` and the most recent stable releases; if your issue affects older releases, please include that information in your report.

## Public Disclosure

Please allow maintainers reasonable time to respond and remediate before publicly disclosing a vulnerability. We follow coordinated disclosure practices and will work with you on CVE assignment when applicable.
